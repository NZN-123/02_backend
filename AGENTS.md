# AGENTS.md

## Git Commit Convention

커밋 메시지는 다음 형식을 사용합니다.

```text
english-category : 한글 메시지
```

예시:

```text
docs : README 링크 구조 정리
fix : 자바스크립트 예제 오류 수정
style : CSS 실습 코드 정리
```

- 카테고리는 영어 소문자를 사용합니다.
- 콜론 앞뒤에는 공백을 둡니다.
- 메시지는 변경 내용을 한글로 간결하게 작성합니다.

## Git Workflow

- 요청받은 파일 변경을 완료하면 관련 파일만 명시적으로 스테이징합니다.
- 커밋 메시지는 위 컨벤션을 따릅니다.
- 커밋 후 현재 브랜치의 원격 추적 브랜치로 푸시합니다.
- 요청 범위 밖의 미추적 파일이나 사용자 변경 파일은 포함하지 않습니다.

## README Guidelines

- 제목은 저장소 목적이 바로 드러나는 한글 제목을 사용합니다.
- 제목 아래에는 저장소의 핵심 기술을 나타내는 flat-square Shields.io 배지를 배치합니다.
- 설명 문장은 개조식으로 간결하게 작성합니다.
- 중복되거나 구조만 보면 알 수 있는 뻔한 설명은 생략합니다.
- 상단에는 `바로가기` 표를 두어 주요 폴더로 이동할 수 있게 합니다.
- `학습 자료`는 차시코드 앞 2자리 기준으로 분류합니다.
- 각 학습 자료 분류는 `<details>`와 `<summary>`를 사용해 접을 수 있게 작성합니다.
- 각 `<summary>` 아래에는 관련 기술 배지를 한 줄로 표시한 뒤 표를 둡니다.
- 학습 자료 표는 `차시`, `주제`, `슬라이드`, `실습` 열을 사용합니다.
- 슬라이드와 실습 경로는 실제 파일 및 폴더로 연결되는 상대 링크를 사용합니다.
- 폴더 규칙에는 `slides/{차시코드 앞 2자리}`와 `exercises/{차시코드 앞 2자리}/{차시코드}` 구조를 명시합니다.

## HTML Docs Guidelines

- `docs/index.html`은 저장소의 학습 자료(README.md) 구성이 변경되거나 업데이트될 때마다 동일하게 갱신해야 합니다.
- 디자인은 Bootstrap 5를 활용하여 반응형 웹 및 세련된 레이아웃을 구현합니다.
- 디자인은 Bootstrap 5의 가장 최신 릴리스 버전을 활용하여 구성합니다.
- 슬라이드와 실습 경로 링크는 `docs/` 경로 기준으로 상대 경로(`../slides/`, `../exercises/`)를 활용하여 작성합니다.
- 단, GitHub Pages 배포 시 웹 루트 외부 폴더(`../`)에 위치한 PDF 슬라이드 및 리소스는 접근 불가능하므로, 슬라이드는 GitHub 원격 파일 뷰 링크(예: `https://github.com/.../blob/main/slides/...`), 실습 코드는 GitHub 원본 코드 뷰 링크(예: `https://github.com/.../tree/main/exercises/...`)로 연결하여 서빙합니다.
